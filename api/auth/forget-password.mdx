---
title: "Reset Password"
openapi: "POST /api/auth/forget-password"
---

The Reset Password API allows users to securely reset forgotten passwords using email verification with an email address, new password, and OTP or verification token.

## Endpoint

`POST {{BASE_URL}}/api/auth/forget-password`

### Request

The request should include the following JSON body:

```json
{
  "email": "user.email@gmail.com",
  "code": 4050,
  "password": "UserPassword@123"
}
```

### Success Response

The Response should include the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Password forget successfully",
  "data": {}
}
```

### Bad Request Responses

Provided Request Server cannot or will not process the request due to something that is perceived to be a client error.

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthorized Responses

If the provided email or password is not found in the database, the response might look like this:

```json
{
  "statusCode": 401,
  "message": "Incorrect credentials.",
  "data": {}
}
```

### Data Not Found Responses

If the provided email or password is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "User not found",
  "data": {}
}
```

### Validation Error

If the request body does not contain a valid email, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Email must be a valid email address."
  "data": {}
}
```

### Server Error

If there is an internal server error, the API responds might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
