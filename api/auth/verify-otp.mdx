---
title: "Verify OTP"
openapi: "POST /auth/verify-otp"
---

The Verify OTP API confirms user identity by validating a one-time password (OTP) sent via the user's registered email during authentication processes.

## Endpoint

`POST {{BASE_URL}}/api/auth/verify-otp`

### Request

The request should include the following JSON body:

```json
{
  "email": "user.email@gmail.com",
  "code": 7845
}
```

### Success Response

The response should include the following JSON body:

```json
{
  "statusCode": 200,
  "message": "OTP verified successfully",
  "data": {}
}
```

### Bad Request Responses

If the server cannot or will not process the request due to something perceived to be a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Data Not Found Responses

If the provided code is incorrect, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Incorrect code entered. Please enter again.",
  "data": {}
}
```

### Validation Error

If the request body does not contain a valid email, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Email must be a valid email address.",
  "data": {}
}
```

### Server Error

If there is an internal server error, the API response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
