---
title: "Verify OTP"
openapi: "POST /auth/verify-otp"
---

Verify OTP API confirms user identity by validating a one-time password (OTP) sent via user's registered email during authentication processes.

## Endpoint

`POST {{BASE_URL}}/api/auth/verify-otp`

### Request

The request should include the following JSON body:

```json
{
  "email": "user.email@gmail.com",
  "code": 7845
}
```

### Success Response

The Response should include the following JSON body:

```json
{
  "statusCode": 200,
  "message": "OTP verify successfully",
  "data": {}
}
```

### Bad Request Responses

Provided Request Server cannot or will not process the request due to something that is perceived to be a client error.

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthorized Responses

If the provided email or password is not found in the database, the response might look like this:

```json
{
  "statusCode": 401,
  "message": "Incorrect code entered. Please enter again.",
  "data": {}
}
```

### Validation Error

If the request body does not contain a valid email, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Email must be a valid email address."
  "data": {}
}
```

### Server Error

If there's an internal server error, the response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
