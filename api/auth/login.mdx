---
title: "Login"
openapi: "POST /auth/login"
---

This API endpoint is used for user authentication. It verifies the provided email and password, and if the credentials are correct, it returns a token or session information to the client.

## Endpoint

`POST {{BASE_URL}}/api/auth/login`

### Request

The request should include the following JSON body:

```json
{
  "email": "user.email@gmail.com",
  "password": "UserPassword@123"
}
```

### Bad Request Responses

Provided Request Server cannot or will not process the request due to something that is perceived to be a client error.

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthorized Responses

If the provided email or password is incorrect, the response might look like this:

```json
{
  "statusCode": 401,
  "message": "Incorrect credentials.",
  "data": {}
}
```

### Data Not Found Responses

If the provided email is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Not Found: User does not exist",
  "data": {}
}
```

### Validation Error

If the request body does not contain a valid email, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Email must be a valid email address."
  "data": {}
}
```

### Server Error

If there is an internal server error, the API response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
