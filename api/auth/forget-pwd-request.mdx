---
title: "Forget Password Request"
openapi: "POST /api/auth/forget-pwd-request"
---

The Forget Password Request API endpoint initiates the password recovery process by sending one-time password (OTP) to the user's registered email. It requires the user's email address and returns a success message if the one-time password (OTP) are sent successfully.

## Endpoint

`POST {{BASE_URL}}/api/auth/forget-pwd-request`

### Request

The request should include the following JSON body:

```json
{
  "email": "jenishcodezee@gmail.com"
}
```

### Success Response

The response should include the following JSON body for a successful request:

```json
{
  "statusCode": 200,
  "message": "OTP send successfully",
  "data": {}
}
```

### Bad Request Responses

Provided Request Server cannot or will not process the request due to something that is perceived to be a client error.

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Error Responses

If the provided email is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "User not found",
  "data": {}
}
```

### Validation Error

If the request body does not contain a valid email, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Email must be a valid email address."
}
```

### Server Error

If there's an internal server error, the response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
