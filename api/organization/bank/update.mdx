---
title: "Update"
openapi: "PUT /api/organization/bank/{bankId}"
---

The Company Bank Update API is designed to handle requests for updating the details of an existing company bank in the system. This API is part of the organization management module and is responsible for modifying company bank details in the database.

## Endpoint

`PUT {{BASE_URL}}/api/organization/bank/{bankId}`

### Request

The request should include the following JSON body:

```json
{
  "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
  "bank_id": "af56f283-5e99-4a32-818e-97056f1f4f37",
  "branch": "KORMANGALA",
  "ifsc_code": "HDFC0001234",
  "account_number": "6543210987654321"
}
```

### Success Response

On successful update of the company bank, the API responds with the following JSON body:

```json
{

  "data": {

    "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",

    "branch": "KORMANGALA",
    "ifsc_code": "HDFC0001234",
    "account_number": "6543210987654321",
    "is_deleted": false,
    "created_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",

    "updated_at": "2024-06-17T07:00:29.788Z",
    "deleted_at": null
  }
}
```

### Bad Request Responses

Provided Request Server cannot or will not process the request due to something that is perceived to be a client error.

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API responds might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the provided company bank is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Company bank not found.",
  "data": {}
}
```

### Validation Error

If the request body does not contain valid data or required fields are missing, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Please provide a bank name."
}
```

### Server Error

If there is an internal server error, the API responds might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
