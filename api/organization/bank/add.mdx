The Company Bank Creation API is designed to handle requests for registering a new company bank account in the system. This API is part of the organization management module and is responsible for adding company bank details to the database.

## Endpoint

`POST {{BASE_URL}}/api/organization/bank`

### Request

To register a new company bank account, the client must send a POST request to the specified endpoint with the following JSON payload:

```json
{
  "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
  "bank_id": "af56f283-5e99-4a32-818e-97056f1f4f37",
  "branch": "KORMANGALA",
  "ifsc_code": "HDFC0001234",
  "account_number": "6543210987654321"
}
```

### Success Response

On successful creation of a company bank account, the API responds with a status code of 201 and the following JSON body:

```json
{
  "statusCode": 201,
  "message": "Company bank added successfully.",
  "data": {
    "id": "30c76755-a22f-42bd-a266-882efb045d78",

```

### Bad Request Responses

Provided Request Server cannot or will not process the request due to something that is perceived to be a client error.

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API responds might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the provided company bank account is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Company bank account not found.",
  "data": {}
}
```

### Validation Error

If the request body does not contain valid data, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Please provide a valid account number."
}
```

### Server Error

If there is an internal server error, the API responds might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
