title: "Update"
openapi: "PUT /api/organization/company/{id}"
---

Company Update API is designed to handle requests for updating existing company details in the system. This API is part of the organization management module and is responsible for modifying and saving the updated company information to the database.

## Endpoint

`PUT {{BASE_URL}}/api/organization/company/{id}`

### Request

The request should include the following JSON body:

```json
{
  "company_name": "Code dot infotech",
  "email": "code.dot@infotech.com",
  "mobile_number": "8596000257",
  "director_name": "Code Dot",
  "pan_no": "EFGHE1234I",
  "address": "123 Innovation Street",
  "pin_code": "805500",
  "city": "0f4ce030-f672-49bd-a8b6-ee52d2303a31",

  "country": "cb6c8d39-ab05-4220-90d6-8eda6b3b9ce5"
}
```

### Bad Request Responses

Provided Request Server cannot or will not process the request due to something that is perceived to be a client error.

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the provided company is not found in the database, the response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the provided email or password is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Company not found.",
  "data": {}
}
```

### Validation Error

If the request body does not contain a valid email, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Please provide a company name."
}
```

### Server Error

If there's an internal server error, the response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
