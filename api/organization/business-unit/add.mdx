The Business unit Creation API is designed to handle requests for registering a new business-unit in the system. This API is part of the organization management module and is responsible for adding business-unit details to the database.

## Endpoint

`POST {{BASE_URL}}/api/organization/business-unit`

### Request

To register a new business-unit, the client must send a POST request to the specified endpoint with the following JSON payload:

```json
{
  "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
  "business_unit_name": "Code Info",
  "email": "",
  "phone_number": "",
  "mobile_number": "",
  "address": "",
  "city": null,
  "state": null,
  "country": null,
  "pincode": ""
}
```

### Bad Request Responses

Provided Request Server cannot or will not process the request due to something that is perceived to be a client error.

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API responds might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the provided Business unit not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Business unit not found.",
  "data": {}
}
```

### Validation Error

If the request body does not contain a valid email, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Please provide a business-unit name."
}
```

### Server Error

If there is an internal server error, the API responds with a status code of 500:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
