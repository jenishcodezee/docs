The List Employee Codes API allows clients to retrieve a list of employee codes based on specified criteria. This API is part of the settings management module and provides access to employee code information stored in the database.

## Endpoint

`POST {{BASE_URL}}/api/settings/employee-code/list`

### Request

To retrieve a list of employee codes, send a POST request to the endpoint with the following JSON payload:

```json
{
  "page": 1,
  "limit": 15,
  // "is_active": true,
  "search": ""
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Invalid required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API responds with the following JSON body:

```json
{
  "statusCode": 401,
  "message": "You are an unauthorized user",
  "data": {}
}
```

### Validation Error

If the request body contains invalid data, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Limit must be greater than or equal to 1"
}
```

### Server Error

If there is an internal server error, the API responds might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
