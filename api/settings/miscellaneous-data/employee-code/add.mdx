The Employee Code Creation API allows clients to register a new employee code within the system. This API is part of the settings management module and is responsible for storing employee code details in the database.

## Endpoint

`POST {{BASE_URL}}/api/settings/employee-code`

### Request

To register a new employee code, send a POST request to the endpoint with the following JSON payload:

```json
{
  "name": "New Logic Build PVT LTD",
  "series_start": 1,
  "is_active": true, // true, false
  "prefix": "NLBC",
  "suffix": "C",
  "numeric_series": 1
}
```

### Success Response

On successful creation of an employee code, the API responds with the following JSON body:

```json
{
  "statusCode": 201,
  "message": "Employee code added successfully.",
  "data": {
    "id": "01ef8c49-1597-420c-bbc7-3d6c1e234ae9",
    "name": "New Logic Build PVT LTD",
    "series_start": 1,
    "prefix": "NLC",
    "suffix": "C",
    "numeric_series": 1,
    "is_active": true,
    "is_deleted": false,
    "created_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "updated_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "deleted_by": null,
    "created_at": "2024-06-18T12:06:29.343Z",
    "updated_at": "2024-06-18T12:06:29.343Z",
    "deleted_at": null,

    "last_created_number": 1
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API responds with the following JSON body:

```json
{
  "statusCode": 401,
  "message": "You are an unauthorized user",
  "data": {}
}
```

### Data Not Found Responses

If the provided employee code is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Employee code not found.",
  "data": {}
}
```

### Validation Error

If the request body does not contain valid data, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Prefix can be a maximum of 3 characters."
}
```

### Server Error

If there is an internal server error, the API response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
