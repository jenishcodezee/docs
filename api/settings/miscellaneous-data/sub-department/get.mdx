---
title: "Get"
openapi: "GET /api/settings/sub-department/{subDepartmentId}"
---

The Sub-department Retrieval API is designed to handle requests for fetching the details of a specific sub-department in the system. This API is part of the settings management module and is responsible for retrieving sub-department details from the database.

## Endpoint

`GET {{BASE_URL}}/api/settings/sub-department/{subDepartmentId}`

### Success Response

On successful retrieval of the sub-department, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Sub-department retrieved successfully.",
  "data": {
    "id": "5fa85f88-6cd3-4020-8d76-eaaa231d9b2d",
    "department_id": "149ff3db-3d87-4756-950b-3c8621dbdd08",
    "department_name": "HR35",
    "sub_department_code": "SM",
    "sub_department_name": "Software Manager",
    "description": "This is the Software Manager.",
    "is_active": false,
    "action_at": "2024-06-17T11:12:49.389Z",
    "action_by": "Jenish Chanchad"
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Not found Responses

If the specified sub-department is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Sub-department not found.",
  "data": {}
}
```

### Server Error

If there is an internal server error, the response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
