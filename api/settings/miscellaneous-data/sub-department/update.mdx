---
title: "Update"
openapi: "PUT /api/settings/sub-department/{subDepartmentId}"
---

The Sub-department Update API allows clients to update details of an existing sub-department within a department. This API is part of the settings management module and is responsible for modifying sub-department information in the database.

## Endpoint

`PUT {{BASE_URL}}/api/settings/sub-department/{subDepartmentId}`

### Request

To update a sub-department, send a PUT request to the endpoint with the following JSON payload:

```json
{
  "department_id": "cda5b942-8c91-4a62-9f67-4721732eac9c",
  "sub_department_code": "SM",
  "sub_department_name": "Software Manager",
  "description": "This is the Software Manager.",
  "is_active": false
}
```

### Success Response

On successful update of the sub-department, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Sub-department updated successfully.",
  "data": {
    "id": "5fa85f88-6cd3-4020-8d76-eaaa231d9b2d",
    "department_id": "cda5b942-8c91-4a62-9f67-4721732eac9c",
    "sub_department_code": "SM",
    "sub_department_name": "Software Manager",
    "description": "This is the Software Manager.",
    "is_active": false,
    "is_deleted": false,
    "created_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "updated_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "created_at": "2024-06-17T11:12:49.389Z",
    "updated_at": "2024-06-17T11:12:49.389Z",
    "deleted_by": null,
    "deleted_at": null
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API responds might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the specified sub-department ID is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Sub-department not found.",
  "data": {}
}
```

### Validation Error

If the request body does not contain valid data or required fields are missing, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Sub-department code already exists.",
  "data": {}
}
```

### Server Error

If there is an internal server error, the API responds might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
