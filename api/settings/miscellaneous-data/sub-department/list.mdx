The Sub-department Listing API allows clients to retrieve a list of sub-departments based on specified criteria. This API is part of the settings management module and supports filtering and pagination for efficient data retrieval.

## Endpoint

`POST {{BASE_URL}}/api/settings/sub-department/list`

### Request

To retrieve a list of sub-departments, send a POST request to the endpoint with optional query parameters:

```json
{
  "page": 1,
  "limit": 15,
  // "is_active": true,
  "search": ""
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Validation Error

If the request body contains invalid data, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Limit must be greater than or equal to 1"
}
```

### Server Error

If there is an internal server error, the API responds might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
