---
title: "Delete"
openapi: "DELETE /api/settings/sub-department/{subDepartmentId}"
---

The Sub-department Deletion API is designed to handle requests for deleting an existing sub-department within a department from the system. This API is part of the settings management module and is responsible for removing sub-department details from the database.

## Endpoint

`DELETE {{BASE_URL}}/api/settings/sub-department/{subDepartmentId}`

### Success Response

On successful deletion of a sub-department, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Sub-department deleted successfully.",
  "data": {}
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the specified sub-department is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Sub-department not found.",
  "data": {}
}
```

### Server Error

If there is an internal server error, the response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
