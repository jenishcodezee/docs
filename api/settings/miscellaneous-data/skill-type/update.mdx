---
title: "Update"
openapi: "PUT /api/settings/skill-type/{skillTypeId}"
---

The Skill Type Update API allows clients to update details of an existing skill type within the system. This API is part of the settings management module and is responsible for modifying skill type information in the database.

## Endpoint

`PUT {{BASE_URL}}/api/settings/skill-type/{skillTypeId}`

### Request

To update a skill type, send a PUT request to the endpoint with the following JSON payload:

```json
{
  "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
  "skill_type_name": "Highly Skilled",
  "skill_type_code": "HSKL",
  "description": "Highly skilled worker",
  "is_active": true
}
```

### Success Response

On successful update of the skill type, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Skill type updated successfully.",
  "data": {
    "id": "07159a0a-1805-438f-9ed2-69bde3acd24b",
    "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
    "skill_type_code": "HSKL",
    "skill_type_name": "Highly Skilled",
    "description": "Highly skilled worker",
    "is_active": true,
    "is_deleted": false,
    "created_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "updated_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "deleted_by": null,
    "created_at": "2024-06-18T11:24:19.488Z",
    "updated_at": "2024-06-18T11:49:48.094Z",
    "deleted_at": null
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API responds with the following JSON body:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the specified skill type ID is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Skill type not found.",
  "data": {}
}
```

### Validation Error

If the request body does not contain valid data, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "This Skill type name or code is already in use",
  "data": {}
}
```

### Server Error

If there is an internal server error, the API response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
