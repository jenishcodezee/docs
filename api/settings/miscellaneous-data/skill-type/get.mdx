---
title: "Get"
openapi: "GET /api/settings/skill-type/{skillTypeId}"
---

The Skill Type Get Single API allows clients to retrieve details of a specific skill type within the system. This API is part of the settings management module and is responsible for fetching skill type information from the database.

## Endpoint

`GET {{BASE_URL}}/api/settings/skill-type/{skillTypeId}`

### Success Response

On successful retrieval of the skill type, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Skill type retrieved successfully.",
  "data": {
    "id": "07159a0a-1805-438f-9ed2-69bde3acd24b",
    "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
    "skill_type_code": "HSKL",
    "skill_type_name": "Highly Skilled",
    "description": "Highly skilled worker",
    "is_active": true,
    "company_name": "CMCREATION PVT LTD",
    "action_by": "Jenish Chanchad",
    "action_at": "2024-06-18T11:49:48.094Z"
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "You are an unauthorized user",
  "data": {}
}
```

### Not found Responses

If the specified skill type ID is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Skill type not found.",
  "data": {}
}
```

### Server Error

If there is an internal server error, the response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
