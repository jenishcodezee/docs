---
title: "Update"
openapi: "PUT /api/settings/industry/{industryId}"
---

The Industry Update API allows clients to update details of an existing industry within the system. This API is part of the settings management module and is responsible for modifying industry information in the database.

## Endpoint

`PUT {{BASE_URL}}/api/settings/industry/{industryId}`

### Request

To update an industry, send a PUT request to the endpoint with the following JSON payload:

```json
{
  "industry_code": "IT",
  "industry_name": "Information Technology",
  "description": "This is the updated description for Information Technology industry.",
  "is_active": true
}
```

### Success Response

On successful update of the industry, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Industry updated successfully.",
  "data": {
    "id": "98efd683-9301-4451-9b61-db14316e2d26",
    "industry_name": "Node.js",
    "is_deleted": false,
    "created_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "updated_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "deleted_by": null,
    "created_at": "2024-06-18T04:50:07.916Z",
    "updated_at": "2024-06-18T05:28:40.327Z",
    "deleted_at": null,
    "description": "This is Node js technology",
    "is_active": true
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the specified industry is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Industry not found.",
  "data": {}
}
```

### Validation Error

If the request body does not contain valid data, such as a duplicate industry code, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Please provide an industry name."
}
```

### Server Error

If there is an internal server error, the API response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
