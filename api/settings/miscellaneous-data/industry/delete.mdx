---
title: "Delete"
openapi: "DELETE /api/settings/industry/{industryId}"
---

The Industry Deletion API is designed to handle requests for deleting an existing industry from the system. This API is part of the settings management module and is responsible for removing industry details from the database.

## Endpoint

`DELETE {{BASE_URL}}/api/settings/industry/{industryId}`

### Success Response

On successful deletion of a industry, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Industry deleted successfully.",
  "data": {}
}
```

### Bad Request Responses

If the server cannot process the request due to a client error (e.g., invalid industry ID format), the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API responds with a status code of 401:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the specified industry ID is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Industry not found.",
  "data": {}
}
```

### Server Error

If there is an internal server error, the API responds with a status code of 500:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
