The List Banks API allows clients to retrieve a list of banks from the system. This API is part of the settings management module and is responsible for fetching bank details based on specified criteria.

## Endpoint

`POST {{BASE_URL}}/api/settings/banks/list`

### Request

To retrieve a list of banks, send a POST request to the endpoint with optional query parameters for filtering:

```json
{
  "page": 1,
  "limit": 15,
  // "is_active": true,
  "search": "sTAte"
}
```

### Success Response

On successful retrieval of banks, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Banks listed successfully.",
  "data": {
    "totalPages": 1,
    "currentPage": 1,
    "totalCount": 1,
    "banks": [
      {
        "id": "d015301a-a4bc-475c-9d72-9a7c2b384f50",
        "bank_code": "SBI",
        "bank_name": "State Bank of India",
        "description": "State Bank of India",
        "is_active": true,
        "action_by": "Chirag S",
        "action_at": "2024-06-07T06:50:41.977Z",
        "full_count": "1"
      }
    ]
  }
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Validation Error

If the request body contains invalid data, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Limit must be greater than or equal to 1"
}
```

### Server Error

If there is an internal server error, the API responds might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
