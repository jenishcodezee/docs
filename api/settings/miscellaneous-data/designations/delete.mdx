---
title: "Delete"
openapi: "DELETE /api/settings/designation/{designationId}"
---

The Designation Deletion API allows clients to delete an existing designation within the company from the system. This API is part of the settings management module and is responsible for removing designation details from the database.

## Endpoint

`DELETE {{BASE_URL}}/api/settings/designation/{designationId}`

### Success Response

On successful deletion of a designation, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Designation deleted successfully.",
  "data": {}
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the specified designation is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Designation not found.",
  "data": {}
}
```

### Server Error

If there is an internal server error, the response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
