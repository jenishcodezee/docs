title: "Get"
openapi: "GET /api/settings/designation/{designationId}"
---

The Designation Retrieval API is designed to handle requests for fetching details of a specific designation in the system. This API is part of the settings management module and is responsible for retrieving designation details from the database.

## Endpoint

`GET {{BASE_URL}}/api/settings/designation/{designationId}`

### Success Response

On successful retrieval of the designation, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Designation retrieved successfully.",
  "data": {
    "id": "6bb486d5-7108-4c18-b715-5786cdec7aff",
    "designation_code": "PRM",
    "designation_name": "Project Reposter",
    "description": "This is example of Project Reposter.",
    "is_active": true,
    "action_at": "2024-06-17T12:15:10.984Z",
    "action_by": "Jenish Chanchad"
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Not found Responses

If the specified designation is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Designation not found.",
  "data": {}
}
```

### Server Error

If there is an internal server error, the response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
