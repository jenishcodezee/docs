---
title: "Update"
openapi: "PUT /api/settings/grade/{gradeId}"
---

The Grade Update API allows clients to update details of an existing grade within the system. This API is part of the settings management module and is responsible for modifying grade information in the database.

## Endpoint

`PUT {{BASE_URL}}/api/settings/grade/{gradeId}`

### Request

To update a grade, send a PUT request to the endpoint with the following JSON payload:

```json
{
  "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
  "grade_name": "Senior Manager Officer",
  "grade_code": "AD",
  "description": "Senior Manager Officer",
  "is_active": true
}
```

### Success Response

On successful update of the grade, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Grade updated successfully.",
  "data": {
    "id": "2d1f11b6-c9d9-4183-b765-ca8e3781cd83",
    "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
    "grade_code": "AD",
    "grade_name": "Senior Manager Officer",
    "description": "Senior Manager Officer",
    "is_active": true,
    "is_deleted": false,
    "created_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "updated_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "deleted_by": null,
    "created_at": "2024-06-18T07:24:21.908Z",
    "updated_at": "2024-06-18T07:35:14.104Z",
    "deleted_at": null
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the specified grade ID is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Grade not found.",
  "data": {}
}
```

### Validation Error

If the request body does not contain valid data, such as a duplicate grade code, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "This grade name is already in use",
  "data": {}
}
```

### Server Error

If there is an internal server error, the API response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
