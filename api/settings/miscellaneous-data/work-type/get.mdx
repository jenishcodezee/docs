---
title: "Get"
openapi: "GET /api/settings/work-type/{workTypeId}"
---

The Work-type Retrieval API allows clients to fetch details of a specific work-type within the company's organizational structure. This API is part of the settings management module and is responsible for retrieving work-type details from the database.

## Endpoint

`GET {{BASE_URL}}/api/settings/work-type/{workTypeId}`

### Success Response

On successful retrieval of a work-type, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Work type retrieved successfully.",
  "data": {
    "id": "caef9389-83ec-4bd8-813e-6e835031e829",
    "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
    "work_type_code": "PA",
    "work_type_name": "Part-Time",
    "description": "Part-Time",
    "is_active": true,
    "company_name": "CMCREATION PVT LTD",
    "action_by": "Jenish Chanchad",
    "action_at": "2024-06-18T10:39:07.484Z"
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Not found Responses

If the provided work type ID is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Work-type not found.",
  "data": {}
}
```

### Server Error

If there is an internal server error, the response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
