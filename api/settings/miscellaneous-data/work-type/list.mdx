The Work-type List API allows clients to retrieve a list of work-types within the company's organizational structure. This API is part of the settings management module and is responsible for fetching work-type details from the database.

## Endpoint

`POST {{BASE_URL}}/api/settings/work-type/list`

### Request

To retrieve a list of work-types, send a POST request to the endpoint with the following JSON payload. This payload can include filters and pagination options as required.

```json
{
  "page": 1,
  "limit": 15,
  // "is_active": true,
  "search": ""
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Validation Error

If the request body contains invalid data, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Limit must be greater than or equal to 1"
}
```

### Server Error

If there is an internal server error, the API responds might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
