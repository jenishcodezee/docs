title: "List"

The Department Listing API retrieves a paginated list of departments within the organization based on specified filters. It is used for managing organizational departments and fetching details from the database.

## Endpoint

`POST {{BASE_URL}}/api/settings/department/list`

### Request

To retrieve department information, send a POST request with the following JSON payload:

```json
{
  "page": 1,
  "limit": 15,
  "is_active": true,
  "search": "",
  "filters": {
    "department_name": "HR",
    "company_name": "cMcReATiOn"
  },
  "responseFields": ["id", "department_name", "company_name"]
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Validation Error

If the request body contains invalid data, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Limit must be greater than or equal to 1"
}
```

### Server Error

If there is an internal server error, the API responds might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
