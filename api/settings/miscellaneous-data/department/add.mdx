The Department Creation API is designed to handle requests for registering a new department within a company in the system. This API is part of the settings management module and is responsible for adding department details to the database.

## Endpoint

`POST {{BASE_URL}}/api/settings/department`

### Request

To register a new department, the client must send a POST request to the specified endpoint with the following JSON payload:

```json
{
  "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
  "department_code": "SM",
  "department_name": "Sales and Marketing",
  "description": "This is Accounting department.",
  "is_active": false
}
```

### Success Response

On successful creation of a department, the API responds with a status code of 201 and the following JSON body:

```json
{
  "statusCode": 201,
  "message": "Department added successfully.",
  "data": {
    "id": "9b1d4f7e-9c9a-4f7e-bb2e-1c4e9f10b2f4",
    "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
    "department_code": "SM",
    "department_name": "Sales and Marketing",
    "description": "This is Accounting department.",
    "is_active": false,
    "created_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "updated_by": "128605d9-1ac6-4e96-9915-51f5bc3749a6",
    "created_at": "2024-06-17T06:40:48.011Z",
    "updated_at": "2024-06-17T06:40:48.011Z"
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API responds might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Data Not Found Responses

If the provided department is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Company not found.",
  "data": {}
}
```

### Validation Error

If the request body does not contain valid data, the response might include a validation error:

```json
{
  "statusCode": 409,
  "message": "Validation Error",
  "data": "Please provide a valid department code."
}
```

### Server Error

If there is an internal server error, the API response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
