---

The Department Retrieval API is designed to handle requests for fetching the details of a specific department in the system. This API is part of the settings management module and is responsible for retrieving department details from the database.

## Endpoint

`GET {{BASE_URL}}/api/settings/department/{departmentId}`

### Success Response

On successful retrieval of the department, the API responds with the following JSON body:

```json
{
  "statusCode": 200,
  "message": "Department retrieved successfully.",
  "data": {
    "id": "fd2f40f8-dd1f-4fbe-8c56-4d2894582b6f",
    "company_id": "816a2506-91c6-4b23-841d-cd174e42a074",
    "company_name": "CMCREATION PVT LTD",
    "department_code": "SM",
    "department_name": "Sales and Marketing",
    "description": "This is Accounting department.",
    "is_active": false,
    "action_at": "2024-06-17T09:23:57.462Z",
    "action_by": "Jenish Chanchad"
  }
}
```

### Bad Request Responses

If the server cannot process the request due to a client error, the response might look like this:

```json
{
  "statusCode": 400,
  "message": "Bad Request: Missing required fields",
  "data": {}
}
```

### Unauthenticated User Responses

If the request lacks proper authentication, the API response might look like this:

```json
{
  "statusCode": 401,
  "message": "Token required",
  "data": {}
}
```

### Not found Responses

If the specified company is not found in the database, the response might look like this:

```json
{
  "statusCode": 404,
  "message": "Company not found.",
  "data": {}
}
```

### Server Error

If there is an internal server error, the response might look like this:

```json
{
  "statusCode": 500,
  "message": "Internal server error. Please try again later.",
  "data": {}
}
```
